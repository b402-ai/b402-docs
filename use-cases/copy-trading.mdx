## Overview

Zara Agents is a copy trading platform that lets followers mirror leader strategies with built-in SLA enforcement and verifiable execution receipts. All follower trades are gasless, non-custodial, and protected by automatic refunds if execution quality falls below agreed thresholds.

## What You'll Build

A complete copy trading system that:
- Enables non-custodial following of trading strategies
- Provides gasless execution for all followers
- Issues ERC-8004 NFT receipts for every trade
- Enforces SLA limits (max slippage, max latency)
- Auto-refunds followers for SLA breaches
- Charges performance fees only on verified profits

## Architecture

```
┌──────────────────────────────────────────────┐
│ FOLLOWER                                      │
│ Signs spend cap: dailyCapUSDT, maxSlippageBps│
│                  maxLatencyMs via B402        │
└───────────────┬──────────────────────────────┘
                │
                ▼
┌──────────────────────────────────────────────┐
│ LEADER TRADES                                 │
│ Leader executes → webhook to Zara server     │
└───────────────┬──────────────────────────────┘
                │
                ▼
┌──────────────────────────────────────────────┐
│ MIRROR & TRACK                                │
│ • Record leader price & timestamp             │
│ • Execute follower trade via PancakeSwap V3  │
│ • Calculate latency & slippage                │
│ • Mint ERC-8004 receipt with verification    │
└───────────────┬──────────────────────────────┘
                │
                ▼
┌──────────────────────────────────────────────┐
│ SETTLEMENT                                    │
│ • Calculate PnL from verified receipts       │
│ • IF slippage/latency > max: auto-refund     │
│ • ELSE IF PnL > 0: charge 10% + 0.20% fee   │
└──────────────────────────────────────────────┘
```

## Implementation

### 1. Register as a Leader

```typescript
// POST /leader/register
const response = await fetch('http://localhost:3003/leader/register', {
  method: 'POST',
  headers: { 'Content-Type': 'application/json' },
  body: JSON.stringify({
    address: '0xLeaderAddress',
    name: 'Strategy Alpha',
    description: 'High-frequency stablecoin arbitrage on BSC'
  })
});

const { leaderId, status } = await response.json();
console.log(`Leader registered: ${leaderId}`);
```

### 2. Authorize Following (Follower)

```typescript
// Step 1: Start authorization flow
const authStart = await fetch('http://localhost:3003/auth/start', {
  method: 'POST',
  headers: { 'Content-Type': 'application/json' },
  body: JSON.stringify({
    follower: followerAddress,
    leaderId: 'leader-uuid',
    dailyCapUSDT: 25,        // Max $25/day
    maxSlippageBps: 30,      // Max 0.3% slippage
    maxLatencyMs: 1500       // Max 1.5 second delay
  })
});

const { eip712Payload } = await authStart.json();

// Step 2: Sign B402 authorization
const signature = await wallet.signTypedData(
  eip712Payload.domain,
  eip712Payload.types,
  eip712Payload.message
);

// Step 3: Complete authorization
await fetch('http://localhost:3003/auth/complete', {
  method: 'POST',
  headers: { 'Content-Type': 'application/json' },
  body: JSON.stringify({
    follower: followerAddress,
    leaderId: 'leader-uuid',
    dailyCapUSDT: 25,
    maxSlippageBps: 30,
    maxLatencyMs: 1500,
    signature
  })
});

console.log('✅ Following enabled! Trades will be mirrored with SLA protection.');
```

### 3. Execute Leader Trade

```typescript
// Leader executes trade and sends webhook
await fetch('http://localhost:3003/lead/fill', {
  method: 'POST',
  headers: { 'Content-Type': 'application/json' },
  body: JSON.stringify({
    leader: leaderAddress,
    tokenIn: '0x55d398326f99059fF775485246999027B3197955', // USDT
    tokenOut: '0xbb4CdB9CBd36B01bD1cBaEBF2De08d9173bc095c', // WBNB
    amountIn: ethers.parseUnits('10', 18).toString(),
    price: '1.0002',
    txHash: '0xLeaderTxHash...'
  })
});

// Server automatically mirrors trade to all followers
```

### 4. View Trade Receipts

```typescript
// Get all receipts for an address
const receipts = await fetch(
  `http://localhost:3003/receipts?address=${followerAddress}`
);

const trades = await receipts.json();

trades.forEach(trade => {
  console.log(`Trade #${trade.id}:`);
  console.log(`  Leader price: ${trade.leaderPrice}`);
  console.log(`  Follower price: ${trade.followerPrice}`);
  console.log(`  Latency: ${trade.latencyMs}ms`);
  console.log(`  Slippage: ${trade.slippageBps}bps`);
  console.log(`  SLA breached: ${trade.slaBreached}`);
  console.log(`  ERC-8004 NFT: ${trade.receiptTokenId}`);
  console.log(`  IPFS: ${trade.ipfsUrl}`);
});
```

### 5. Settle and Collect Fees

```typescript
// Settle last 24 hours of trades
await fetch('http://localhost:3003/settle/period', {
  method: 'POST',
  headers: { 'Content-Type': 'application/json' },
  body: JSON.stringify({
    followerId: 'follower-uuid',
    leaderAddress: leaderAddress
  })
});

// Server calculates:
// - Total PnL from verified receipts
// - Issues refunds for SLA-breached trades
// - Charges 10% performance fee on profits
// - Charges 0.20% execution fee on all trades
```

## API Reference

### Leader Management

#### `POST /leader/register`

Register a new trading strategy.

```typescript
interface RegisterLeaderRequest {
  address: string;
  name: string;
  description: string;
}

interface RegisterLeaderResponse {
  leaderId: string;
  status: string;
}
```

### Follower Authorization

#### `POST /auth/start`

Start authorization flow for following a leader.

```typescript
interface AuthStartRequest {
  follower: string;
  leaderId: string;
  dailyCapUSDT: number;
  maxSlippageBps: number;
  maxLatencyMs: number;
}

interface AuthStartResponse {
  eip712Payload: {
    domain: any;
    types: any;
    message: any;
  };
```

Complete authorization after signing.

```typescript
interface AuthCompleteRequest {
  follower: string;
  leaderId: string;
  dailyCapUSDT: number;
  maxSlippageBps: number;
```

Notify server of leader trade (mirrors to followers).


## SLA Enforcement

Automatic protection for followers:

<Steps>
  <Step title="Track Execution Quality">
    Server records exact leader price, timestamp, and follower execution data
  </Step>

  <Step title="Calculate Metrics">
    Latency = follower timestamp - leader timestamp
    Slippage = price deviation in basis points
  </Step>

  <Step title="Check Against Limits">
    If latency > maxLatencyMs OR slippage > maxSlippageBps:
    - Trade marked as `slaBreached: true`
    - Breach reason recorded on-chain
  </Step>

  <Step title="Issue Refunds">
    During settlement:
    - Follower receives refund = execution fee + slippage delta
    - No performance fee charged
  </Step>
</Steps>

### Best Practices

<Steps>
  <Step title="Wallet Security">
    - Use HSM or KMS for server private key
    - Monitor wallet balance and set alerts
    - Rotate keys periodically
  </Step>

  <Step title="Trade Validation">
    - Verify leader signatures on webhook calls
    - Check follower authorization before execution
    - Validate daily spend caps
  </Step>

  <Step title="Receipt Storage">
    - Pin all receipts to IPFS permanently
    - Store backup copies in database
    - Verify ERC-8004 minting succeeded
  </Step>

  <Step title="Monitoring">
    - Alert on failed trades
    - Track SLA breach rates
    - Monitor settlement errors
  </Step>
</Steps>

## Troubleshooting

<AccordionGroup>
  <Accordion title="Follower authorization failed">
    - Check follower has sufficient balance for daily cap
    - Verify B402 facilitator is accessible
    - Ensure correct network and token addresses
  </Accordion>

  <Accordion title="Trade not mirrored">
    - Verify leader is registered
    - Check follower has active authorization
    - Ensure daily cap not exceeded
    - Review server logs for errors
  </Accordion>

  <Accordion title="Receipt minting failed">
    - Check server wallet has ETH for gas on Sepolia
    - Verify ERC-8004 contract is deployed
    - Ensure Pinata JWT is valid
    - Check IPFS upload succeeded
  </Accordion>

  <Accordion title="Settlement errors">
    - Verify all trades have receipts minted
    - Check follower has sufficient balance for fees
    - Review database for trade data integrity
  </Accordion>
