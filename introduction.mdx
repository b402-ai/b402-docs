b402 is a multichain payment protocol for AI agents and web services. It extends the [HTTP 402](https://datatracker.ietf.org/doc/html/rfc7231#section-6.5.2) standard and [x402](https://x402.org) protocol with gasless execution, compliant confidentiality, and multi-token support across EVM-compatible chains.

## Architecture

Rather than depending on token-level signature standards like EIP-3009, b402 introduces its own execution layer — the Relayer contract — that works with any whitelisted token on any deployed chain. An agent signs an EIP-712 authorization off-chain (zero cost), the Facilitator settles on-chain, and the agent never needs native tokens for gas.

b402 is organized in three layers:

1. **Protocol Layer** — The b402 Relayer smart contract handles EIP-712 signature verification and on-chain token transfers. Deployed on BNB Chain and Base, with more chains to follow.
2. **Service Layer** — The Facilitator verifies payment payloads, settles transactions, deploys smart wallets, and executes swaps. b402scan provides transaction discovery.
3. **Developer Layer** — `b402-sdk` (TypeScript) and `b402-express` (middleware) let you integrate in minutes.

## What Makes b402 Different

Compared to standard x402 implementations:

| Feature | x402 (Coinbase) | b402 |
|---------|-----------------|------|
| Tokens | USDC only | USDT, USDC, USD1, any whitelisted ERC-20 |
| Privacy | None | RAILGUN integration (shield, transact, unshield) |
| Wallets | EOA only | EOA + Smart Wallets (ERC-7579 Nexus) |
| Gas | Buyer pays | Gasless for buyers (facilitator covers gas) |
| Swaps | None | Integrated DEX swaps |
| Identity | None | ERC-8004 agent identity |
| Chains | Base (USDC only) | Multichain — BNB Chain, Base, more planned |

## How It Works

```
1. Agent sends GET /premium-data to a b402-enabled server
2. Server responds: 402 Payment Required + payment instructions
3. Agent signs EIP-712 payment payload (no gas needed)
4. Agent resends request with X-PAYMENT header
5. Server calls facilitator /verify then /settle
6. Facilitator executes on-chain transfer
7. Server returns the data
```

## Who Is b402 For?

**Agents and Developers (Buyers)**
AI agents, bots, and scripts that need to autonomously pay for APIs, data, or compute. Smart wallet support means agents can operate with gasless, batched transactions across multiple chains.

**Service Providers (Sellers)**
Developers who want to monetize APIs with crypto payments. Add `b402-express` middleware to your Express app and start charging per-request, no payment gateway needed.

## What Can You Build?

- **Autonomous API trading bots** that pay per query
- **AI agents** purchasing verified data feeds or inference time
- **Usage-based paywalls** for analytics, content, or compute
- **Private payments** where transaction details are shielded via RAILGUN
- **Cross-chain agent workflows** that transact wherever liquidity exists
