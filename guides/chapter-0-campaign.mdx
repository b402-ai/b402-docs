**Chapter 0** is b402's early adopter rewards program. Points are earned from on-chain transaction volume (shield deposits), distributed daily from a fixed weekly emission pool, and amplified through referrals.

## How It Works

1. **Deposit tokens** into b402 via shield operations (RAILGUN privacy pool)
2. **Earn points daily** proportional to your share of the global deposit volume
3. **Refer others** to earn additional points from their deposits
4. **Track progress** via the Chapter 0 stats API and leaderboard

Points accrue automatically from on-chain data. No manual claiming is required.

## Points System

### Emission Model

Points are distributed from a **fixed weekly emission pool** that is split evenly across each day of the week:

```
WEEKLY_EMISSION = 10,000,000 points  (configurable via CHAPTER_0_WEEKLY_EMISSION)
DAILY_EMISSION  = WEEKLY_EMISSION / 7 = ~1,428,571 points per day
```

Each day's emission is divided among all participants proportionally to their deposit volume.

### Daily Point Calculation

For each day, a user's points are the sum of two components:

```
dailyPoints = ownPoints + referralPoints
```

**Own points** (70% weight by default):

```
ownPoints = OWN_WEIGHT * (userDailyDeposits / globalDailyDeposits) * DAILY_EMISSION
```

**Referral points** (30% weight by default):

```
referralPoints = REFERRAL_WEIGHT * (referralDailyDeposits / globalDailyDeposits) * DAILY_EMISSION
```

Where:
- `userDailyDeposits` = total USD value of the user's shield deposits that day
- `referralDailyDeposits` = total USD value of all referred users' shield deposits that day
- `globalDailyDeposits` = total USD value of all shield deposits across the protocol that day

### Configuration Parameters

| Parameter | Default | Env Variable | Description |
|-----------|---------|-------------|-------------|
| Weekly emission | 10,000,000 | `CHAPTER_0_WEEKLY_EMISSION` | Total points distributed per week |
| Own weight | 0.70 (70%) | `CHAPTER_0_OWN_WEIGHT` | Multiplier for user's own deposits |
| Referral weight | 0.30 (30%) | `CHAPTER_0_REFERRAL_WEIGHT` | Multiplier for referral deposits |
| Campaign start | Today | `CHAPTER_0_CAMPAIGN_START_DATE` | ISO date when point accrual begins |

### Example Calculation

Suppose on a given day:
- Global deposits: $100,000
- Your deposits: $5,000 (5% of global)
- Your referrals' deposits: $2,000 (2% of global)
- Daily emission: 1,428,571 points

```
ownPoints      = 0.70 * (5,000 / 100,000) * 1,428,571 = 50,000 points
referralPoints = 0.30 * (2,000 / 100,000) * 1,428,571 =  8,571 points
dailyTotal     = 58,571 points
```

### Key Properties

- **Proportional**: Your share of the daily emission scales linearly with your deposit share
- **Zero-sum per day**: The total points distributed each day equals exactly `DAILY_EMISSION`
- **Consistent usage rewarded**: Depositing every day earns more than one large deposit
- **Weekly window**: Points are tracked on a Monday-to-Sunday week (UTC)

## What Counts as a Deposit

Points are calculated from **shield operations** (deposits into the RAILGUN privacy pool) for supported stablecoin tokens:

| Token | Address (BSC Mainnet) |
|-------|-----------------------|
| USDT | `0x55d398326f99059fF775485246999027B3197955` |
| USDC | `0x8AC76a51cc950d9822D68b83fE1Ad97B32Cd580d` |
| USD1 | `0x8d0d000ee44948fc98c9b98a4fa4921476f08b0d` |

All tokens use 18 decimals on BNB Chain. Amounts are converted from wei to USD at 1:1 (stablecoin peg assumed).

## Referral System

Each participant can have a set of **referred wallet addresses** (referees). When a referred user makes shield deposits, a portion of the daily emission is credited to the referrer.

### How Referrals Work

1. A referrer shares their referral link (managed by the b402 auth service)
2. A new user signs up and their wallet address is linked to the referrer
3. The auth service passes the list of referee addresses when calling the stats API
4. Both own and referral deposits contribute to the referrer's daily points

### Referral Limits

- Up to **1,000 referral wallet addresses** can be tracked per stats request
- Referral data is trusted from the auth service (not re-verified by the backend)

### Referral Tracking in the Response

The stats API returns each referee's all-time deposit total:

```typescript
{
  referrals: [
    { address: "0xReferee1...", totalDeposits: "15000.00" },
    { address: "0xReferee2...", totalDeposits: "8500.50" }
  ]
}
```

## API: Chapter 0 Stats

### POST /users/chapter0-stats

This is the primary endpoint for retrieving a user's Chapter 0 data. It is **admin-only** (requires `X-Admin-Secret` header) and is called by the b402 auth service.

#### Request

```typescript
// POST /users/chapter0-stats
// Headers: { "X-Admin-Secret": "your-admin-secret", "Content-Type": "application/json" }

{
  walletAddress: string,                // User's wallet address (0x...)
  referralWalletAddresses: string[]     // Array of referred users' addresses (max 1000)
}
```

#### Response

```typescript
{
  // All-time shield deposit total (USD value)
  totalUsdValue: "45230.500",

  // Monthly breakdown (last 12 months, newest first)
  monthly: [
    { month: "2026-03-01T00:00:00.000Z", amount: "12500.000" },
    { month: "2026-02-01T00:00:00.000Z", amount: "18200.000" },
    { month: "2026-01-01T00:00:00.000Z", amount: "14530.500" }
  ],

  // Referral data (only referees with deposits)
  referrals: [
    { address: "0xreferee1...", totalDeposits: "15000.00" },
    { address: "0xreferee2...", totalDeposits: "8500.50" }
  ],

  // Chapter 0 points (current week)
  chapter0: {
    weekToDatePoints: "158571.43",
    dailyBreakdown: [
      {
        date: "2026-03-03T00:00:00.000Z",
        ownDeposits: "5000.000000",
        referralDeposits: "2000.000000",
        points: {
          total: "58571.43",
          own: "50000.00",
          referral: "8571.43"
        }
      },
      {
        date: "2026-03-04T00:00:00.000Z",
        ownDeposits: "8000.000000",
        referralDeposits: "3000.000000",
        points: {
          total: "100000.00",
          own: "80000.00",
          referral: "20000.00"
        }
      }
    ]
  }
}
```

#### Response Fields

| Field | Description |
|-------|-------------|
| `totalUsdValue` | All-time total of the user's shield deposits in USD (3 decimal precision) |
| `monthly` | Array of monthly deposit totals for the last 12 months |
| `referrals` | Each referee's all-time deposit total (only includes referees with at least one deposit) |
| `chapter0.weekToDatePoints` | Total points earned so far in the current Monday-Sunday week |
| `chapter0.dailyBreakdown` | Per-day breakdown with own deposits, referral deposits, and point split |

#### Authentication

The endpoint uses the `guardian` middleware which checks the `X-Admin-Secret` header:

```typescript
// The header must match the ADMIN_API_SECRET environment variable (case-insensitive)
headers: {
  "X-Admin-Secret": "your-admin-secret"
}
```

Requests without a valid secret receive a `401 Unauthorized` response.

#### Caching

Responses are cached for **1 hour** per unique combination of wallet address and referral addresses. This reduces database load for repeated queries.

## Data Pipeline

The Chapter 0 data pipeline runs as a seeder job that processes on-chain shield events:

<Frame>
  <img src="/public/images/diagrams/chapter0-pipeline.svg" alt="Chapter 0 data pipeline: on-chain events through seeder to database tables and Stats API" />
</Frame>

The seeder:
1. Reads all shield events since the last sync time
2. Filters for supported stablecoin token addresses
3. Aggregates deposits by wallet address and UTC day
4. Upserts into `WalletDailyDeposits` (per-wallet) and `GlobalDailyTotals` (protocol-wide)
5. Updates the sync cursor to avoid reprocessing

## Leaderboard Integration

The Chapter 0 campaign integrates with the public metrics API for leaderboard display:

```bash
# Top buyers by transaction count
curl "https://backend.b402.ai/metrics/buyers?take=10&skip=0"

# Individual buyer stats
curl "https://backend.b402.ai/metrics/buyers/0xYourAddress..."

# Global protocol metrics (transactions, volume, buyer/seller counts)
curl "https://backend.b402.ai/metrics"
```

The metrics endpoints are public (no authentication required) and provide:
- Transaction count and total volume
- Paginated buyer/seller leaderboards
- Per-address transaction history and volume

## How to Participate

### As a Buyer

1. Set up a wallet (EOA or smart wallet via the [Wallet guide](/concepts/wallet))
2. Fund your wallet with USDT, USDC, or USD1 on BNB Chain
3. Make shield deposits through b402 to start earning points
4. Share your referral link to earn from referrals' deposits

### As a Seller

1. Integrate b402 payment acceptance into your API
2. Your transaction volume is tracked on-chain
3. Shield deposits you make also count toward your Chapter 0 points

### Maximizing Points

- **Deposit consistently**: Daily deposits earn more than sporadic large deposits because you capture a share of each day's emission
- **Grow your referral network**: Referral deposits contribute 30% weight to your points
- **Use supported tokens**: Only USDT, USDC, and USD1 shield deposits count

## Timeline

Chapter 0 is the first phase of the b402 rewards program. The campaign start date is configurable (`CHAPTER_0_CAMPAIGN_START_DATE`). Future chapters will introduce additional incentives, governance participation, and ecosystem rewards.
