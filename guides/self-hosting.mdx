This guide covers everything you need to self-host the b402 facilitator and backend services. The facilitator handles payment verification, UserOp construction, and on-chain settlement. The backend provides analytics and privacy indexing.

## Prerequisites

- **Node.js 18+** with npm or yarn
- **MongoDB** for the facilitator (wallet deployment records, transaction state)
- **PostgreSQL 15+** for the backend (Prisma ORM, privacy indexing)
- **A funded wallet** with BNB for gas (the relayer wallet)
- **An ERC-4337 bundler** endpoint (Biconomy, Pimlico, Stackup, or self-hosted)
- **Docker** (recommended for production)

## Environment Variables: Facilitator

All configuration is done via environment variables. The facilitator validates required variables at startup and fails fast if any are missing.

### Server

| Variable | Required | Default | Description |
|----------|----------|---------|-------------|
| `SERVER_PORT` | No | `3402` | HTTP server port |
| `SERVER_HOST` | No | `0.0.0.0` | Bind address |
| `NETWORK` | No | `testnet` | Network selection: `mainnet` or `testnet` |
| `ADMIN_API_SECRET` | No | -- | Secret for bypassing rate limits (sent as `X-Admin-Secret` header) |
| `DATABASE_URL` | No | -- | MongoDB connection string (e.g., `mongodb://localhost:27017/b402-facilitator`) |

### Blockchain RPC

| Variable | Required | Default | Description |
|----------|----------|---------|-------------|
| `BSC_RPC_URL` | No | `https://bsc-dataseed.binance.org` | BNB Chain mainnet RPC endpoint |
| `BSC_TESTNET_RPC_URL` | No | `https://data-seed-prebsc-1-s1.binance.org:8545` | BNB Chain testnet RPC endpoint |

<Warning>
Public RPC endpoints have rate limits. For production, use a dedicated RPC provider (Ankr, QuickNode, or a private node).
</Warning>

### EOA Payment (Relayer)

| Variable | Required | Default | Description |
|----------|----------|---------|-------------|
| `RELAYER_PRIVATE_KEY` | **Yes** | -- | Private key of the relayer wallet. Pays gas for EOA payment settlement via `transferWithAuthorization()`. |
| `B402_RELAYER_ADDRESS` | **Yes** | -- | Address of the deployed B402 Relayer contract. Mainnet: `0x590162bf4b50F6576a459B75309eE21D92178A10` |

<Warning>
Never commit `RELAYER_PRIVATE_KEY` to source control. Use a secrets manager in production (AWS Secrets Manager, HashiCorp Vault, etc.).
</Warning>

### Smart Wallet (ERC-4337)

| Variable | Required | Default | Description |
|----------|----------|---------|-------------|
| `ENTRY_POINT_ADDRESS` | **Yes** | -- | ERC-4337 EntryPoint v0.7: `0x0000000071727De22E5E9d8BAf0edAc6f37da032` |
| `BUNDLER_RPC_URL` | **Yes** | -- | URL of your ERC-4337 bundler (e.g., `https://bundler.biconomy.io/api/v2/56/...`) |
| `PAYMASTER_ADDRESS` | **Yes** | -- | Address of your deployed paymaster contract |
| `PAYMASTER_SIGNER_KEY` | **Yes** | -- | Private key used to sign paymaster authorizations |
| `WALLET_PAYMENT_FEE` | **Yes** | -- | Fee charged per smart wallet payment (e.g., `0.01` for 1%) |
| `WALLET_DEPLOYMENT_FEE` | **Yes** | -- | Fee charged for smart wallet deployment (in token wei units) |
| `WALLET_FEE_RECIPIENT` | **Yes** | -- | Address that receives protocol fees |

### Nexus Smart Account (ERC-7579)

| Variable | Required | Default | Description |
|----------|----------|---------|-------------|
| `NEXUS_ACCOUNT_FACTORY_ADDRESS` | **Yes** | -- | Nexus factory: `0x0000006648ED9B2B842552BE63Af870bC74af837` |
| `NEXUS_BOOTSTRAP_ADDRESS` | **Yes** | -- | Bootstrap contract: `0x0000003eDf18913c01cBc482C978bBD3D6E8ffA3` |
| `K1_VALIDATOR_ADDRESS` | **Yes** | -- | ECDSA validation module address (K1Validator) |

### Swap

| Variable | Required | Default | Description |
|----------|----------|---------|-------------|
| `SWAP_SERVICE_FEE` | No | `0.01` | Fee percentage for swap operations (0.01 = 1%) |

## Environment Variables: Backend

The backend uses PostgreSQL via Prisma and has its own set of variables.

### Core

| Variable | Required | Description |
|----------|----------|-------------|
| `DATABASE_URL` | **Yes** | PostgreSQL connection string (e.g., `postgresql://user:pass@localhost:5432/b402`) |
| `ADMIN_API_SECRET` | **Yes** | Required for protected endpoints (`/privacy/unshield`) |

### Privacy (RAILGUN)

| Variable | Default | Description |
|----------|---------|-------------|
| `RAILGUN_IMPL` | `official` | RAILGUN implementation: `official` or `custom`. Determines which Relay contract address is used. |

## Minimal .env Files

### Facilitator .env

```bash
# Network
NETWORK=mainnet
BSC_RPC_URL=https://bsc-dataseed1.binance.org

# Relayer (EOA payments)
RELAYER_PRIVATE_KEY=0x_YOUR_RELAYER_PRIVATE_KEY
B402_RELAYER_ADDRESS=0x590162bf4b50F6576a459B75309eE21D92178A10

# ERC-4337 (Smart wallet payments)
ENTRY_POINT_ADDRESS=0x0000000071727De22E5E9d8BAf0edAc6f37da032
BUNDLER_RPC_URL=https://your-bundler-endpoint.com
PAYMASTER_ADDRESS=0x_YOUR_PAYMASTER_ADDRESS
PAYMASTER_SIGNER_KEY=0x_YOUR_PAYMASTER_SIGNER_KEY
WALLET_PAYMENT_FEE=0.01
WALLET_DEPLOYMENT_FEE=1000000000000000000
WALLET_FEE_RECIPIENT=0x_YOUR_FEE_RECIPIENT

# Nexus (ERC-7579)
NEXUS_ACCOUNT_FACTORY_ADDRESS=0x0000006648ED9B2B842552BE63Af870bC74af837
NEXUS_BOOTSTRAP_ADDRESS=0x0000003eDf18913c01cBc482C978bBD3D6E8ffA3
K1_VALIDATOR_ADDRESS=0x_YOUR_K1_VALIDATOR_ADDRESS

# Optional
SWAP_SERVICE_FEE=0.01
DATABASE_URL=mongodb://localhost:27017/b402-facilitator
ADMIN_API_SECRET=your-secret-key-here
```

### Backend .env

```bash
DATABASE_URL=postgresql://b402:your-password@localhost:5432/b402
ADMIN_API_SECRET=your-secret-key-here

# Privacy
RAILGUN_IMPL=official

```

<Warning>
Never commit `.env` files to version control. The `RELAYER_PRIVATE_KEY` and `PAYMASTER_SIGNER_KEY` control wallets that hold funds.
</Warning>

## Running the Facilitator

```bash
# Clone and install
git clone https://github.com/b402-ai/b402-facilitator.git
cd b402-facilitator
npm install

# Start in development
npm run dev

# Start in production
npm run build && npm start
```

The server starts on `http://localhost:3402` by default.

## Database Setup

### MongoDB (Facilitator)

The facilitator uses MongoDB for wallet deployment records and transaction state. No schema migration is needed; collections are created automatically.

```bash
# Verify MongoDB is accessible
mongosh mongodb://localhost:27017/b402-facilitator
> db.stats()
```

### PostgreSQL (Backend)

The backend uses Prisma with PostgreSQL. Run migrations before starting:

```bash
cd b402-backend
npx prisma migrate deploy
```

Key tables created by the migration:

| Table | Purpose |
|-------|---------|
| `AuthorizationTransfers` | EOA payment records indexed from on-chain events |
| `Shields` | RAILGUN shield events (deposits into privacy pool) |
| `Nullifiers` | Spent nullifiers (prevents double-spending) |
| `Commitments` | Merkle tree leaf nodes (commitment hashes) |
| `MerkleTreeNodes` | Full Merkle tree structure (all levels, Poseidon hashes) |
| `PendingUnshield` | Queued unshield requests with scheduled broadcast times |

## Docker Deployment

### Facilitator Dockerfile

```dockerfile
FROM node:18-alpine
WORKDIR /app
COPY package*.json ./
RUN npm ci --production
COPY . .
RUN npm run build
EXPOSE 3402
CMD ["npm", "start"]
```

### Backend Dockerfile

```dockerfile
FROM node:18-alpine
WORKDIR /app
COPY package*.json ./
RUN npm ci --production
COPY . .
RUN npx prisma generate
RUN npm run build
EXPOSE 3000
CMD ["node", "dist/index.js"]
```

### Docker Compose

```yaml
version: "3.8"

services:
  facilitator:
    build:
      context: ./b402-facilitator
    ports:
      - "3402:3402"
    env_file: ./b402-facilitator/.env
    depends_on:
      - mongodb
    restart: unless-stopped

  backend:
    build:
      context: ./b402-backend
    ports:
      - "3000:3000"
    env_file: ./b402-backend/.env
    depends_on:
      - postgres
    restart: unless-stopped

  mongodb:
    image: mongo:7
    ports:
      - "27017:27017"
    volumes:
      - mongo-data:/data/db
    restart: unless-stopped

  postgres:
    image: postgres:16-alpine
    ports:
      - "5432:5432"
    environment:
      POSTGRES_USER: b402
      POSTGRES_PASSWORD: your-secure-password
      POSTGRES_DB: b402
    volumes:
      - pg-data:/var/lib/postgresql/data
    restart: unless-stopped

volumes:
  mongo-data:
  pg-data:
```

```bash
docker compose up -d
```

## Wallet Funding

The relayer wallet needs BNB to pay gas for on-chain transactions:

| Operation | Approximate Gas | BNB Cost (at 3 gwei) |
|-----------|----------------|----------------------|
| EOA payment settlement | ~100,000 | ~0.0003 BNB |
| Smart wallet deployment | ~400,000 | ~0.0012 BNB |
| Smart wallet payment (UserOp) | ~200,000 | ~0.0006 BNB |

**Testnet**: Use the [BNB testnet faucet](https://www.bnbchain.org/en/testnet-faucet).

**Mainnet**: Fund the relayer wallet with at least **0.5 BNB** initially. Set up balance monitoring to alert when it drops below 0.1 BNB.

For smart wallet payments, gas is covered by the paymaster. The paymaster contract must be funded with BNB, and the `PAYMASTER_SIGNER_KEY` must be authorized on the contract.

## Security Hardening

### Private Key Management

- Store `RELAYER_PRIVATE_KEY` and `PAYMASTER_SIGNER_KEY` in a secrets manager (AWS Secrets Manager, HashiCorp Vault)
- Never log private keys, even in development
- Use separate keys for testnet and mainnet
- Rotate keys periodically and immediately after any potential exposure

### Network Security

- Run behind a reverse proxy (nginx, Caddy) with TLS termination
- Restrict MongoDB and PostgreSQL to internal network access only
- Use firewall rules to limit access to service ports

```nginx
# nginx rate limiting example
limit_req_zone $binary_remote_addr zone=b402:10m rate=10r/s;

server {
    location /api/ {
        limit_req zone=b402 burst=20 nodelay;
        proxy_pass http://localhost:3402;
    }
}
```

### Rate Limiting

The backend includes built-in rate limiting:

- **Default**: 30 requests per minute per IP
- **Admin bypass**: Requests with a valid `X-Admin-Secret` header skip rate limiting
- Configure stricter limits at the reverse proxy level for production

### Admin Secret

- Set a strong, random `ADMIN_API_SECRET` (at least 32 characters)
- This secret protects the `/privacy/unshield` endpoint
- Share it only with trusted services (e.g., the auth service that calls the backend)

### Monitoring Checklist

Set up alerts for:
- Relayer wallet BNB balance below threshold
- Failed UserOp submissions (bundler errors)
- Database connection failures
- High error rates on `/settle` endpoints
- Unusual transaction volume or patterns
