## GET /api/v1/health

Returns the current status of the Facilitator service, including the active network and relayer address.

### Authentication

Not required.

### Request

No headers, query parameters, or body required.

### Response

#### Success (200)

| Field | Type | Description |
|-------|------|-------------|
| `status` | `string` | Service status. `"ok"` when healthy. |
| `service` | `string` | Service identifier. Always `"b402-facilitator"`. |
| `network` | `string` | Active blockchain network (`"bsc"` or `"base"`). |
| `relayer` | `string` | Relayer contract address used for settlements. |

#### Example

```json
{
  "status": "ok",
  "service": "b402-facilitator",
  "network": "bsc",
  "relayer": "0xE91b564EB8DFF305Ff8efA332f84c487b9da5171"
}
```

### Errors

| Code | Status | Description |
|------|--------|-------------|
| `internal_server_error` | 500 | Service is unavailable or misconfigured |

### Code Examples

#### cURL

```bash
curl -X GET https://facilitatorv3.b402.ai/api/v1/health
```

#### TypeScript

```typescript
const response = await fetch("https://facilitatorv3.b402.ai/api/v1/health");
const data = await response.json();

console.log(data.status);   // "ok"
console.log(data.network);  // "bsc"
console.log(data.relayer);  // "0xE91b564EB8DFF305Ff8efA332f84c487b9da5171"
```

<Tip>
Use this endpoint to verify the Facilitator is reachable and to confirm the active network and relayer address before submitting payments.
</Tip>
