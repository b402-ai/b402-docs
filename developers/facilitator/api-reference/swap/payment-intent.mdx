---
title: 'Swap Payment Intent'
description: 'Get EIP-712 payload for a swap transaction'
---

`POST /swap/payment-intent` returns the EIP-712 typed data that the client must sign to authorize a swap payment. This is the first step in the EOA swap flow: get the payload, sign it client-side, then submit it to [`POST /swap`](/developers/facilitator/api-reference/swap/execute).

```
POST /swap/payment-intent
```

## Request Body

| Field | Type | Required | Description |
|-------|------|----------|-------------|
| `from` | `string` | Yes | Payer wallet address |
| `to` | `string` | Yes | Recipient wallet address |
| `token` | `string` | Yes | ERC-20 token contract address |
| `amount` | `string` | Yes | Payment amount in wei |
| `swapParams` | `object` | Yes | Swap configuration |
| `swapParams.tokenIn` | `string` | Yes | Input token contract address |
| `swapParams.tokenOut` | `string` | Yes | Output token contract address |
| `swapParams.amountIn` | `string` | Yes | Input amount in wei |
| `swapParams.minAmountOut` | `string` | Yes | Minimum acceptable output in wei |

## Response

### 200 - EIP-712 Payload

| Field | Type | Description |
|-------|------|-------------|
| `authorization` | `object` | `TransferWithAuthorization` fields to sign |
| `authorization.from` | `string` | Payer wallet address |
| `authorization.to` | `string` | Recipient wallet address |
| `authorization.value` | `string` | Amount in wei |
| `authorization.validAfter` | `number` | Unix timestamp, signature valid after |
| `authorization.validBefore` | `number` | Unix timestamp, signature valid before |
| `authorization.nonce` | `string` | Random bytes32 nonce |
| `domain` | `object` | EIP-712 domain separator |
| `types` | `object` | EIP-712 type definitions |

### Error Responses

| Status | Code | Description |
|--------|------|-------------|
| 400 | `validation_error` | Missing or invalid fields |

## Examples

### cURL

```bash
curl -X POST https://facilitator.b402.ai/swap/payment-intent \
  -H "Content-Type: application/json" \
  -d '{
    "from": "0xAbC1234567890aBcDeF1234567890AbCdEf123456",
    "to": "0x9876543210FeDcBa9876543210FeDcBa98765432",
    "token": "0x55d398326f99059fF775485246999027B3197955",
    "amount": "10000000000000000000",
    "swapParams": {
      "tokenIn": "0x55d398326f99059fF775485246999027B3197955",
      "tokenOut": "0xbb4CdB9CBd36B01bD1cBaEBF2De08d9173bc095c",
      "amountIn": "10000000000000000000",
      "minAmountOut": "15000000000000000"
    }
  }'
```

### TypeScript

```typescript
// 1. Get the EIP-712 payload
const response = await fetch('https://facilitator.b402.ai/swap/payment-intent', {
  method: 'POST',
  headers: { 'Content-Type': 'application/json' },
  body: JSON.stringify({
    from: wallet.address,
    to: recipientAddress,
    token: '0x55d398326f99059fF775485246999027B3197955',
    amount: '10000000000000000000',
    swapParams: {
      tokenIn: '0x55d398326f99059fF775485246999027B3197955',
      tokenOut: '0xbb4CdB9CBd36B01bD1cBaEBF2De08d9173bc095c',
      amountIn: '10000000000000000000',
      minAmountOut: '15000000000000000',
    },
  }),
});

const { authorization, domain, types } = await response.json();

// 2. Sign the EIP-712 typed data
const signature = await wallet.signTypedData(domain, types, authorization);

// 3. Submit to POST /swap with the signed payload
const swapRes = await fetch('https://facilitator.b402.ai/swap', {
  method: 'POST',
  headers: { 'Content-Type': 'application/json' },
  body: JSON.stringify({
    paymentPayload: {
      token: '0x55d398326f99059fF775485246999027B3197955',
      payload: { authorization, signature },
    },
    swapParams: {
      tokenIn: '0x55d398326f99059fF775485246999027B3197955',
      tokenOut: '0xbb4CdB9CBd36B01bD1cBaEBF2De08d9173bc095c',
      amountIn: '10000000000000000000',
      minAmountOut: '15000000000000000',
      deadline: Math.floor(Date.now() / 1000) + 3600,
    },
  }),
});

const result = await swapRes.json();
console.log('Swap result:', result);
```

## Notes

- The returned `authorization` object includes server-generated `validAfter`, `validBefore`, and `nonce` fields
- The `domain` and `types` fields are required for correct EIP-712 signature construction
- Sign the payload client-side using `signTypedData` (ethers.js) or equivalent
