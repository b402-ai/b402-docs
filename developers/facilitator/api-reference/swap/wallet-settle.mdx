---
title: 'Settle Wallet Swap'
description: 'Execute swap via bundler for smart wallet'
---

`POST /swap/wallet/settle` takes the signed UserOperation from [`/swap/wallet/verify`](/developers/facilitator/api-reference/swap/wallet-verify) and submits it to the ERC-4337 bundler. The bundler executes the swap on PancakeSwap V3 and returns the transaction hash with swap results.

```
POST /swap/wallet/settle
```

<Note>
Settlement typically completes in ~3 seconds on BNB Chain. The facilitator covers all gas costs.
</Note>

## Request Body

| Field | Type | Required | Description |
|-------|------|----------|-------------|
| `userOp` | `object` | Yes | The full UserOperation from `/swap/wallet/verify`, with `signature` populated |
| `signature` | `string` | Yes | User's signature of the `userOpHash` (hex) |

The `userOp` object must contain all fields exactly as returned by `/swap/wallet/verify`. Do not modify any field other than `signature`.

## Response

### 200 - Swap Settled

| Field | Type | Description |
|-------|------|-------------|
| `success` | `boolean` | Whether the swap was executed successfully |
| `txHash` | `string` | On-chain transaction hash |
| `userOpHash` | `string` | UserOperation hash matching the one from `/swap/wallet/verify` |
| `swapResult` | `object` | Swap execution details |
| `swapResult.amountIn` | `string` | Actual input amount in wei |
| `swapResult.amountOut` | `string` | Actual output amount in wei |

### Error Responses

| Status | Code | Description |
|--------|------|-------------|
| 400 | `validation_error` | Missing or malformed `userOp` or `signature` |
| 400 | `validation_error` | Signature does not match the `userOpHash` |
| 400 | `userop_error` | Bundler rejected the UserOp (insufficient gas, nonce conflict, paymaster denial) |
| 400 | `swap_error` | Swap reverted (slippage exceeded `minAmountOut`, deadline passed) |
| 500 | `blockchain_error` | Transaction reverted or receipt polling timed out |

## What Happens Server-Side

1. **Injects signature** - Places the user's signature into the UserOp
2. **Submits to bundler** - Sends the signed UserOp via `eth_sendUserOperation`
3. **Polls for receipt** - Waits for the bundler to mine the transaction
4. **Extracts swap result** - Parses swap events from transaction logs
5. **Returns result** - Responds with the transaction hash and swap amounts

## Examples

### cURL

```bash
curl -X POST https://facilitator.b402.ai/swap/wallet/settle \
  -H "Content-Type: application/json" \
  -d '{
    "userOp": {
      "sender": "0x...wallet_address",
      "nonce": "0x...",
      "callData": "0x...encoded_swap_call",
      "callGasLimit": "0x...",
      "verificationGasLimit": "0x...",
      "preVerificationGas": "0x...",
      "maxFeePerGas": "0x...",
      "maxPriorityFeePerGas": "0x...",
      "paymaster": "0x...",
      "paymasterVerificationGasLimit": "0x...",
      "paymasterPostOpGasLimit": "0x...",
      "paymasterData": "0x...",
      "signature": "0x...user_signature_of_userOpHash"
    },
    "signature": "0x...user_signature_of_userOpHash"
  }'
```

### TypeScript

```typescript
import { Wallet } from 'ethers';

// 1. Get unsigned UserOp from /swap/wallet/verify
const verifyRes = await fetch('https://facilitator.b402.ai/swap/wallet/verify', {
  method: 'POST',
  headers: { 'Content-Type': 'application/json' },
  body: JSON.stringify({
    walletAddress: '0x...nexus_smart_wallet',
    swapParams: {
      tokenIn: '0x55d398326f99059fF775485246999027B3197955',
      tokenOut: '0xbb4CdB9CBd36B01bD1cBaEBF2De08d9173bc095c',
      amountIn: '10000000000000000000',
      minAmountOut: '15000000000000000',
    },
    paymentRequirements: { network: 'bsc' },
  }),
});
const { userOp, userOpHash, quote } = await verifyRes.json();

console.log('Expected output:', quote.amountOut);

// 2. Sign the userOpHash
const ownerWallet = new Wallet(process.env.PRIVATE_KEY);
const signature = await ownerWallet.signMessage(
  Buffer.from(userOpHash.slice(2), 'hex')
);

// 3. Submit signed UserOp
const settleRes = await fetch('https://facilitator.b402.ai/swap/wallet/settle', {
  method: 'POST',
  headers: { 'Content-Type': 'application/json' },
  body: JSON.stringify({
    userOp: { ...userOp, signature },
    signature,
  }),
});

const result = await settleRes.json();

if (result.success) {
  console.log('Swap settled');
  console.log('Amount in:', result.swapResult.amountIn);
  console.log('Amount out:', result.swapResult.amountOut);
  console.log('TX:', result.txHash);
  console.log(`View on BscScan: https://bscscan.com/tx/${result.txHash}`);
} else {
  console.error('Settlement failed');
}
```

## Error Handling

Bundler-level failures return `userop_error`. Swap-specific failures return `swap_error`.

| Cause | What To Do |
|-------|------------|
| Nonce already used | Re-call `/swap/wallet/verify` for a fresh UserOp |
| Paymaster expired | Re-call `/swap/wallet/verify` for new paymaster data |
| Slippage exceeded | Increase `minAmountOut` tolerance or retry with a fresh quote |
| Deadline passed | Re-call `/swap/wallet/verify` with a new deadline |
| Insufficient balance | Token balance changed between verify and settle |

For all error responses, the safe recovery path is to re-call `/swap/wallet/verify` and restart the flow.

## Full Flow

```
Client                          Facilitator                    Bundler / PancakeSwap
  |                                  |                                |
  |  POST /swap/wallet/verify       |                                |
  |  { walletAddress, swapParams }  |                                |
  |--------------------------------->|                                |
  |                                  |-- validate wallet, balance     |
  |                                  |-- fetch PancakeSwap quote      |
  |                                  |-- build callData + fee         |
  |                                  |-- estimate gas via bundler --->|
  |                                  |<--- gas estimates -------------|
  |                                  |-- sign paymaster data          |
  |  { userOp, userOpHash, quote }  |                                |
  |<---------------------------------|                                |
  |                                  |                                |
  |  sign(userOpHash)                |                                |
  |                                  |                                |
  |  POST /swap/wallet/settle       |                                |
  |  { userOp + signature }         |                                |
  |--------------------------------->|                                |
  |                                  |-- eth_sendUserOperation ------>|
  |                                  |-- poll for receipt ----------->|
  |                                  |<--- receipt + swap logs -------|
  |  { txHash, swapResult }         |                                |
  |<---------------------------------|                                |
```
