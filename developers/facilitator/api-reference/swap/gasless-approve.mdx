---
title: 'Gasless Approve'
description: 'EIP-2612 gasless token approval for swaps'
---

`POST /swap/gasless-approve` submits an EIP-2612 permit signature to approve token spending without the user paying gas. The facilitator submits the `permit()` transaction on-chain and covers the gas cost.

```
POST /swap/gasless-approve
```

## Request Body

| Field | Type | Required | Description |
|-------|------|----------|-------------|
| `owner` | `string` | Yes | Token owner address (the approver) |
| `spender` | `string` | Yes | Address being approved to spend tokens |
| `token` | `string` | Yes | ERC-20 token contract address (must support EIP-2612) |
| `value` | `string` | Yes | Approval amount in wei |
| `deadline` | `number` | Yes | Unix timestamp after which the permit expires |
| `signature` | `string` | Yes | EIP-2612 permit signature (hex) |

## Response

### 200 - Approval Submitted

| Field | Type | Description |
|-------|------|-------------|
| `success` | `boolean` | Whether the approval transaction was mined successfully |
| `txHash` | `string` | On-chain transaction hash for the `permit()` call |

### Error Responses

| Status | Code | Description |
|--------|------|-------------|
| 400 | `validation_error` | Missing or invalid fields |
| 400 | `signature_error` | Permit signature is invalid or recovery failed |
| 400 | `permit_error` | Token does not support EIP-2612 or permit was already used |
| 500 | `blockchain_error` | Transaction reverted or RPC failure |

## Examples

### cURL

```bash
curl -X POST https://facilitator.b402.ai/swap/gasless-approve \
  -H "Content-Type: application/json" \
  -d '{
    "owner": "0xAbC1234567890aBcDeF1234567890AbCdEf123456",
    "spender": "0x590162bf4b50F6576a459B75309eE21D92178A10",
    "token": "0x55d398326f99059fF775485246999027B3197955",
    "value": "115792089237316195423570985008687907853269984665640564039457584007913129639935",
    "deadline": 1709254800,
    "signature": "0xabc123...eip2612_permit_signature"
  }'
```

### TypeScript

```typescript
import { ethers } from 'ethers';

const tokenAddress = '0x55d398326f99059fF775485246999027B3197955';
const spender = '0x590162bf4b50F6576a459B75309eE21D92178A10';
const value = ethers.MaxUint256.toString();
const deadline = Math.floor(Date.now() / 1000) + 3600;

// 1. Build the EIP-2612 permit signature
const domain = {
  name: 'Tether USD',
  version: '1',
  chainId: 56,
  verifyingContract: tokenAddress,
};

const types = {
  Permit: [
    { name: 'owner', type: 'address' },
    { name: 'spender', type: 'address' },
    { name: 'value', type: 'uint256' },
    { name: 'nonce', type: 'uint256' },
    { name: 'deadline', type: 'uint256' },
  ],
};

const permitData = {
  owner: wallet.address,
  spender,
  value,
  nonce: 0, // fetch from token contract: token.nonces(owner)
  deadline,
};

const signature = await wallet.signTypedData(domain, types, permitData);

// 2. Submit gasless approval
const response = await fetch('https://facilitator.b402.ai/swap/gasless-approve', {
  method: 'POST',
  headers: { 'Content-Type': 'application/json' },
  body: JSON.stringify({
    owner: wallet.address,
    spender,
    token: tokenAddress,
    value,
    deadline,
    signature,
  }),
});

const result = await response.json();

if (result.success) {
  console.log('Gasless approval complete');
  console.log('TX:', result.txHash);
} else {
  console.error('Approval failed');
}
```

## Notes

- The token must support EIP-2612 (`permit()` function). Not all ERC-20 tokens implement this standard.
- The facilitator pays gas for the `permit()` transaction on behalf of the user.
- Approval is a prerequisite for swap execution if the token has not already been approved for the spender.
- For maximum approval, use `type(uint256).max` as the value.
