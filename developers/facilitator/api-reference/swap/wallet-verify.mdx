---
title: 'Verify Wallet Swap'
description: 'Build a swap UserOp for smart wallet execution'
---

`POST /swap/wallet/verify` builds an unsigned ERC-4337 UserOperation that executes a token swap on PancakeSwap V3 through a smart wallet. The facilitator validates the wallet, fetches a quote, injects the fee, estimates gas, and returns a UserOp ready for signing.

```
POST /swap/wallet/verify
```

## Request Body

| Field | Type | Required | Description |
|-------|------|----------|-------------|
| `walletAddress` | `string` | Yes | Nexus smart wallet address (must already be deployed) |
| `swapParams` | `object` | Yes | Swap configuration |
| `swapParams.tokenIn` | `string` | Yes | Input token contract address |
| `swapParams.tokenOut` | `string` | Yes | Output token contract address |
| `swapParams.amountIn` | `string` | Yes | Input amount in wei |
| `swapParams.minAmountOut` | `string` | Yes | Minimum acceptable output in wei (slippage protection) |
| `paymentRequirements` | `object` | Yes | Payment configuration |
| `paymentRequirements.network` | `string` | Yes | Target network (`bsc`) |

## Response

### 200 - Unsigned UserOp with Quote

| Field | Type | Description |
|-------|------|-------------|
| `isValid` | `boolean` | Whether the swap operation can proceed |
| `userOp` | `object` | Unsigned ERC-4337 UserOperation |
| `userOpHash` | `string` | Hash of the UserOp that the client must sign |
| `quote` | `object` | PancakeSwap V3 quote for the swap |
| `quote.amountOut` | `string` | Expected output amount in wei |
| `quote.priceImpact` | `string` | Price impact as a percentage |

### Error Responses

| Status | Code | Description |
|--------|------|-------------|
| 400 | `validation_error` | Missing or invalid fields |
| 400 | `validation_error` | Wallet not deployed or insufficient balance |
| 404 | `no_route` | No liquidity route found for the token pair |
| 500 | `blockchain_error` | Failed to read on-chain state or estimate gas |

## What Happens Server-Side

1. **Validates wallet** - Confirms a deployed contract exists at `walletAddress`
2. **Checks balance** - Verifies the wallet holds enough `tokenIn`
3. **Fetches quote** - Gets the current PancakeSwap V3 price for the pair
4. **Builds UserOp** - Encodes the swap interaction into `callData` with fee injection
5. **Estimates gas** - Calls the ERC-4337 bundler for gas estimates
6. **Signs paymaster data** - The paymaster authorizes gas sponsorship
7. **Returns unsigned UserOp** - Client signs `userOpHash` and submits via [`/swap/wallet/settle`](/developers/facilitator/api-reference/swap/wallet-settle)

<Warning>
Set `minAmountOut` to protect against slippage. Use [`GET /swap/quote`](/developers/facilitator/api-reference/swap/quote) to determine a reasonable value, then apply your desired slippage tolerance.
</Warning>

## Examples

### cURL

```bash
curl -X POST https://facilitator.b402.ai/swap/wallet/verify \
  -H "Content-Type: application/json" \
  -d '{
    "walletAddress": "0x...nexus_smart_wallet",
    "swapParams": {
      "tokenIn": "0x55d398326f99059fF775485246999027B3197955",
      "tokenOut": "0xbb4CdB9CBd36B01bD1cBaEBF2De08d9173bc095c",
      "amountIn": "10000000000000000000",
      "minAmountOut": "15000000000000000"
    },
    "paymentRequirements": {
      "network": "bsc"
    }
  }'
```

### TypeScript

```typescript
const response = await fetch('https://facilitator.b402.ai/swap/wallet/verify', {
  method: 'POST',
  headers: { 'Content-Type': 'application/json' },
  body: JSON.stringify({
    walletAddress: '0x...nexus_smart_wallet',
    swapParams: {
      tokenIn: '0x55d398326f99059fF775485246999027B3197955',  // USDT
      tokenOut: '0xbb4CdB9CBd36B01bD1cBaEBF2De08d9173bc095c', // WBNB
      amountIn: '10000000000000000000', // 10 USDT
      minAmountOut: '15000000000000000', // ~0.015 WBNB
    },
    paymentRequirements: {
      network: 'bsc',
    },
  }),
});

const data = await response.json();

if (data.isValid) {
  console.log('Swap UserOp built');
  console.log('Expected output:', data.quote.amountOut);
  console.log('Price impact:', data.quote.priceImpact, '%');
  console.log('UserOp hash to sign:', data.userOpHash);
  // Sign userOpHash and submit to /swap/wallet/settle
} else {
  console.error('Verification failed');
}
```

## Notes

- PancakeSwap V3 integration on BNB Chain
- Supports any token pair with liquidity on PancakeSwap V3
- The response includes a live quote so the client can display expected output before signing
- The facilitator covers all gas costs via the paymaster

## Next Step

After receiving the response, sign the `userOpHash` with the wallet owner's private key and submit the signed UserOp to [`POST /swap/wallet/settle`](/developers/facilitator/api-reference/swap/wallet-settle).
