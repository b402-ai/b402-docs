---
title: 'Check Nullifiers'
description: 'Check if nullifiers have been spent'
---

`POST /privacy/nullifiers` checks whether nullifiers have been used (spent) on-chain. Returns separate lists of used and unused nullifiers. No authentication required.

```
POST /privacy/nullifiers
```

## Request Body

| Field | Type | Description |
|-------|------|-------------|
| `nullifiers` | `string[]` | Array of nullifier hashes (`0x...`). Min 1, max 1000. |

## Response

### 200 - Nullifier Status

| Field | Type | Description |
|-------|------|-------------|
| `used` | `array` | Nullifiers that have been spent |
| `used[].nullifier` | `string` | The nullifier hash |
| `used[].transactionHash` | `string` | Transaction that spent the nullifier |
| `used[].blockNumber` | `string` | Block number of the spend |
| `used[].blockTime` | `string` | ISO 8601 timestamp of the block |
| `used[].type` | `string` | `"TRANSACT"` or `"UNSHIELD"` |
| `used[].outputCommitments` | `array` | New commitments created by the spend |
| `used[].outputCommitments[].commitmentHash` | `string` | Output commitment hash |
| `used[].outputCommitments[].position` | `number` | Tree position of the output |
| `used[].outputCommitments[].ciphertext` | `object` | Encrypted note data |
| `used[].unshieldData` | `object` | Present only when `type` is `"UNSHIELD"` |
| `used[].unshieldData.recipient` | `string` | Unshield recipient address |
| `used[].unshieldData.token` | `string` | Unshielded token address |
| `used[].unshieldData.amount` | `string` | Unshielded amount in wei |
| `unused` | `string[]` | Nullifier hashes that have not been spent |

### Error Responses

| Status | Code | Description |
|--------|------|-------------|
| 400 | `validation_error` | Missing `nullifiers` array, empty array, exceeds 1000 items, or invalid hex strings |

## Examples

### cURL

```bash
curl -X POST https://backend.b402.ai/privacy/nullifiers \
  -H "Content-Type: application/json" \
  -d '{
    "nullifiers": [
      "0x1234567890abcdef1234567890abcdef1234567890abcdef1234567890abcdef",
      "0xabcdef1234567890abcdef1234567890abcdef1234567890abcdef1234567890"
    ]
  }'
```

### TypeScript

```typescript
const response = await fetch('https://backend.b402.ai/privacy/nullifiers', {
  method: 'POST',
  headers: { 'Content-Type': 'application/json' },
  body: JSON.stringify({
    nullifiers: [
      '0x1234567890abcdef1234567890abcdef1234567890abcdef1234567890abcdef',
      '0xabcdef1234567890abcdef1234567890abcdef1234567890abcdef1234567890',
    ],
  }),
});

const { used, unused } = await response.json();

for (const spent of used) {
  console.log('Spent:', spent.nullifier);
  console.log('Type:', spent.type);
  console.log('Tx:', spent.transactionHash);

  if (spent.type === 'UNSHIELD') {
    console.log('Recipient:', spent.unshieldData.recipient);
    console.log('Amount:', spent.unshieldData.amount);
  }
}

console.log('Unspent nullifiers:', unused.length);
```

## Notes

- Accepts up to 1000 nullifiers per request. For larger sets, batch into multiple calls.
- The `unshieldData` field is only present on nullifiers with `type: "UNSHIELD"`. For `TRANSACT` type, the spend created new shielded commitments listed in `outputCommitments`.
- Responses are not cached.
