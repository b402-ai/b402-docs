---
title: 'Request Unshield'
description: 'Schedule an unshield (privacy withdrawal) operation'
---

`POST /privacy/unshield` schedules a withdrawal from the RAILGUN privacy pool. The unshield is delayed by a random 1-3 hour window for privacy. Requires admin authentication.

```
POST /privacy/unshield
```

<Warning>
This endpoint requires the `X-Admin-Secret` header. Requests without a valid secret return `401 Unauthorized`.
</Warning>

## Request Body

| Field | Type | Description |
|-------|------|-------------|
| `txidVersion` | `string` | RAILGUN transaction ID version |
| `transactionTo` | `string` | Must be the Railgun Relay contract address |
| `transactionData` | `string` | Encoded transaction calldata (`0x...`, max 100KB) |
| `chain` | `object` | Target chain identifier |
| `chain.type` | `number` | Chain type (`0` = EVM) |
| `chain.id` | `number` | Chain ID (e.g., `56` for BNB Chain) |
| `nullifierHash` | `string` | Nullifier hash for the commitment being unshielded |
| `recipientOwnerAddress` | `string` | Recipient Ethereum address |
| `recipientSalt` | `string` | Salt for recipient derivation |

## Response

### 200 - New Request

Returned when this nullifier has not been submitted before.

| Field | Type | Description |
|-------|------|-------------|
| `exists` | `boolean` | `false` |
| `unshieldId` | `string` | UUID for tracking this request |
| `scheduledUnshieldTime` | `string` | ISO 8601 timestamp when the unshield will be broadcast (1-3 hours from now) |

### 200 - Existing Request

Returned when this nullifier was already submitted. No duplicate is created.

| Field | Type | Description |
|-------|------|-------------|
| `exists` | `boolean` | `true` |
| `unshieldId` | `string` | UUID of the existing request |
| `timeElapsedMs` | `number` | Milliseconds since the request was created |
| `scheduledUnshieldTime` | `string` | ISO 8601 scheduled broadcast time |
| `broadcasterTxHash` | `string` | Transaction hash if already broadcast (may be absent) |
| `createdAt` | `string` | ISO 8601 creation timestamp |
| `broadcastedAt` | `string` | ISO 8601 broadcast timestamp (may be absent) |
| `completedAt` | `string` | ISO 8601 completion timestamp (may be absent) |

### Error Responses

| Status | Code | Description |
|--------|------|-------------|
| 400 | `validation_error` | Invalid request body, transaction data exceeds 100KB, or `transactionTo` is not the Relay contract |
| 400 | `nullifier_spent` | Nullifier has already been spent on-chain |
| 401 | `unauthorized` | Missing or invalid `X-Admin-Secret` header |

## Examples

### cURL

```bash
curl -X POST https://backend.b402.ai/privacy/unshield \
  -H "Content-Type: application/json" \
  -H "X-Admin-Secret: $ADMIN_SECRET" \
  -d '{
    "txidVersion": "1",
    "transactionTo": "0xRailgunRelayContractAddress",
    "transactionData": "0x...",
    "chain": { "type": 0, "id": 56 },
    "nullifierHash": "0x1234567890abcdef1234567890abcdef1234567890abcdef1234567890abcdef",
    "recipientOwnerAddress": "0xAbC1234567890aBcDeF1234567890AbCdEf123456",
    "recipientSalt": "0x..."
  }'
```

### TypeScript

```typescript
const response = await fetch('https://backend.b402.ai/privacy/unshield', {
  method: 'POST',
  headers: {
    'Content-Type': 'application/json',
    'X-Admin-Secret': process.env.ADMIN_SECRET!,
  },
  body: JSON.stringify({
    txidVersion: '1',
    transactionTo: '0xRailgunRelayContractAddress',
    transactionData: '0x...',
    chain: { type: 0, id: 56 },
    nullifierHash: '0x1234...abcdef',
    recipientOwnerAddress: '0xAbC123...456',
    recipientSalt: '0x...',
  }),
});

const result = await response.json();

if (!result.exists) {
  console.log('Unshield scheduled:', result.unshieldId);
  console.log('Will broadcast at:', result.scheduledUnshieldTime);
} else {
  console.log('Already submitted:', result.unshieldId);
  if (result.broadcasterTxHash) {
    console.log('Broadcast tx:', result.broadcasterTxHash);
  }
}
```

## Notes

- The random 1-3 hour delay prevents timing correlation attacks that could de-anonymize the recipient.
- `transactionTo` must exactly match the Railgun Relay contract for the specified chain. The server rejects requests targeting any other address.
- Transaction data is capped at 100KB. Larger payloads are rejected.
- If the same `nullifierHash` is submitted again, the server returns the existing request instead of creating a duplicate. This is safe to retry.
- Use the [Unshield Status](/developers/backend/privacy/unshield-status) endpoint to poll for broadcast completion.
